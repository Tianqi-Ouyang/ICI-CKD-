NoteAbout this page

This page is a summary of the data management pipeline. Code is hidden across the site and chunks are not executed at render time — they would load RPDR extracts and patient-level data which stay local. The cleaned output (final_master_0114_raw.xlsx + aki_0120.xlsx) is produced locally and feeds analysis.qmd.

Pipeline overview

Stage Inputs Output
1. Demographics & cohort ICI starts registry + RPDR Dem Race, Ethnic_Group, sex, last-known-alive date
2. Pre-baseline labs (within 180 days of ICI) RPDR Lab pre_CRE_180days, pre_HGB_180days, pre_ALB_180days
3. Baseline eGFR Pre-baseline CRE + age + sex eGFR_cre via CKD-EPI 2021 race-free formula
4. Comorbidities & medications on/before ICI RPDR Dia + Med dm, htn, cad, cirrhosis, esrd_kt, ppi, ace_arb, diu, steroids, smoking
5. Post-ICI CKD outcomes Serial CRE after baseline + CKD-EPI ckd_incidence, ckd_progression, eskd_defination_1 (each with 90-day persistence rule)
6. ESRD / KT after ICI RPDR Dia (post-ICI date) esrd_kt_after_ici, esrd_kt_after_ici_date
7. Cohort exclusions ICI date < 2024-03-31; baseline CRE exists; no pre-existing ESRD/KT; baseline eGFR ≥ 10; no RPDR death-date errors
8. ICI class & companion chemo (±30 days of ICI) RPDR Med ICI_Class (PD1/PDL1/CTLA4/Combination), Bevacizumab, Cisplatin, Carboplatin, Pemetrexed, Gemcitabine, Cetuximab, Trastuzumab, VEGF_TKI, nephrotoxic_chemo
9. AKI from serial CRE RPDR Lab (post-ICI CRE) aki, aki_date, aki_days using KDIGO-style rules (CRE > 1.5× median-90d OR +0.3 over lowest-2d)
10. Composite outcomes & competing-risk status All outcomes + death eskd_composite, ckd_composite, status_ckd_cmp (Censored / CKD / Death), time_to_ckd_cmp
11. Yearly eGFR & CKD stage (1–10 years) Yearly median CRE per patient eGFR_CRE_1year…10year, ckd_stage_1year…10year

Key method decisions

  • eGFR formula — CKD-EPI 2021 race-free creatinine-based equation for all eGFR calculations (baseline, yearly, pre/post-event).
  • Outcome definitions — CKD events require the defining condition to hold for ≥ 90 days (persistence rule) before being counted, to exclude transient dips.
  • AKI — post-ICI creatinine compared to both a 90-day pre-window median (KDIGO 1.5× criterion) and a 2-day pre-window minimum (+0.3 mg/dL criterion).
  • Competing risk — death is modeled as a competing event to CKD composite. Patients who die before any CKD event contribute to the Death stratum of status_ckd_cmp.
  • Pre-index windows — all baseline covariates (medications, diagnoses, labs) are restricted to dates on or before ICI_Dose_Date to prevent temporal leakage.

1.8 Derived — Outcomes (CKD / ESKD / Death)

All outcomes require a 90-day persistence rule (the condition must hold continuously ≥ 90 days) unless noted.

In Code Definition
ckd_incidence First sustained (≥90d) episode with pct_drop > 0.30 & eGFR_CRE < 60 & eGFR_CRE_baseline ≥ 60
ckd_incidence_date Start date of the first qualifying episode
ckd_progression First sustained (≥90d) episode with eGFR_CRE < 60 & eGFR_CRE_baseline < 60
eskd_defination_1 First sustained (≥90d) episode with eGFR_CRE < 10
esrd_kt_after_ici 1 if ESRD/KT diagnosis date after ICI_Dose_Date; esrd_kt_after_ici_date = first post-ICI date
eskd_composite esrd_kt_after_ici == 1 \| eskd_defination_1 == 1
eskd_composite_date pmin(esrd_kt_after_ici_date, eskd_defination_1_date)
ckd_composite eskd_composite == 1 \| ckd_incidence == 1 \| ckd_progression == 1

1.9 Derived — Time Variables

In Code Description
last_cre_date Most recent CRE lab date
last_cre_days Days from ICI to last_cre_date
death_days / time_to_death Days from ICI to Date_Of_Death
time_to_death_status 0 = alive/censored, 2 = death
time_to_eskd_composite Days from ICI to eskd_composite_date
time_to_ckd_incidence Days from ICI to ckd_incidence_date
time_to_ckd_progression Days from ICI to ckd_progression_date
time_to_ckd_composite pmin(time_to_eskd_composite, time_to_ckd_incidence, time_to_ckd_progression)

1.10 Derived — Competing Risk (CKD Composite vs Death)

In Code Description
status_ckd_cmp Factor: Censored (0), CKD Composite (1), Death (2)
time_to_ckd_cmp Event time matching status_ckd_cmp (CKD date, death date, or last-CRE date)
time_to_ckd_cmp_year time_to_ckd_cmp / 365
status_ckd_csh Cause-specific coding: 1 if status_ckd_cmp == "CKD Composite", else 0

1.11 Derived — Follow-up & Year-Level Flags

In Code Description
followed_1year … followed_10year 1 if last_cre_days > 365 × N
ckd_incidence_1year … _10year 1 if time_to_ckd_incidence < 365 × N
ckd_progression_1year … _10year Same pattern
eskd_composite_1year … _10year Same pattern
ckd_composite_1year … _10year Same pattern

1.12 Derived — AKI

In Code Definition
aki 1 if post-ICI CRE exceeds 1.5× median-CRE-in-prior-90-days OR exceeds lowest-CRE-in-prior-2-days by ≥ 0.3 mg/dL
aki_date First post-ICI date meeting the AKI condition
aki_days Days from ICI to aki_date
aki_before_ckd 1 if aki_days <= time_to_ckd_composite

Previous version only applied Medication_Date <= index_date when excluded_names was supplied; the else branch dropped the date guard, so exposures like ppi, ace_arb, and diu could include medications started AFTER ICI. This contaminated “baseline” covariates with post-ICI treatment history and biased downstream models. Date filter is now applied in both branches. Any final_master_*.xlsx built from the old version must be regenerated before use.
